Grand Line Dev

A rota do Lucas até o primeiro emprego dev: 4 fases, 2 trilhas em paralelo (Web/JS + Java da faculdade) e 1 diferencial (IA aplicada). Marque as metas — seu progresso fica salvo.

Leia primeiroComo esta rota funciona

Você navega com duas trilhas ao mesmo tempo, e isso é proposital:

  • JS/WEB Sua trilha de portfólio e velocidade: JavaScript → React → Node → SQL → TypeScript. É onde você coloca projeto no ar rápido e onde estão muitas vagas júnior.
  • JAVA Sua trilha da faculdade e do mercado corporativo: a facul vai te dar lógica e POO em Java — não lute contra isso, capriche nisso. Java domina bancos, fintechs e grandes empresas no Brasil. Quem sabe Java + Spring Boot entra num mercado que paga bem e tem menos júnior brigando por vaga.
  • IA O diferencial: construir com IA (APIs de LLM, RAG, agentes), não só usar ChatGPT. Pouquíssimo júnior sabe. É o que faz recrutador parar de rolar a tela.
  • SOFT O multiplicador: inglês diário + LinkedIn ativo + escrever sobre o que aprende. Sem isso, as outras trilhas rendem metade.

Por que JS + Java juntos não é loucura

Porque os fundamentos são os mesmos: lógica, POO, estrutura de dados, HTTP, banco. Você aprende o conceito uma vez e só muda a sintaxe. A facul puxa Java de qualquer jeito (provas e projetos), então a trilha Java aqui está sincronizada com o que você já vai ter que estudar — zero esforço desperdiçado. E ter as duas stacks no currículo conta a história perfeita: "faço front e entrego rápido com JS, e construo back-end corporativo com Java/Spring".

Regras do navio

  • Constância > intensidade. 5h num dia bom, 1h num dia ruim — mas nunca zero. Commit todo dia.
  • Nunca cole código que não consegue explicar. Em entrevista isso afunda o navio.
  • Projeto inacabado não existe pro recrutador. Melhor 1 projeto pequeno no ar do que 3 pela metade.
  • A facul é o mínimo, não o teto. Use-a pro diploma, pras bases e pra trilha Java. O resto está aqui.

☕ Java na faculdade — Fase 1

O que a facul vai cobrar: lógica, sintaxe Java, POO (classes, objetos, herança, polimorfismo, encapsulamento, interfaces). Sua jogada: não estude "pra prova" — estude pra dominar, porque POO é a base de TUDO (inclusive do JS avançado e do React). Se a aula da facul for fraca, complemente:

Curso em Vídeo — Java Básico (Guanabara) Curso em Vídeo — Java POO DevDojo — Maratona Java (a mais completa do YouTube BR) Loiane Groner — cursos Java grátis

Truque: quando aprender um conceito em Java na facul (ex: herança), escreva no mesmo dia um exemplo do mesmo conceito em JS. Você fixa o conceito 2x e aprende a separar "ideia" de "sintaxe".

🤖 IA nesta fase — modo aprendizado

Use IA como professor particular, não como estagiário: peça pra explicar linha a linha, pedir exercícios sobre o que estudou, e criticar seu código ("aponte 3 problemas neste código e por quê"). Proibido: colar código que não explica.

Fase 3 · meses 8–12Java Pro + IA — o tesouro do diferencial

Objetivo: aqui você deixa de ser "mais um estudando JS" e vira raro: um júnior com Spring Boot (mercado corporativo) e um projeto de IA aplicada funcionando. Esta é a fase que paga a viagem inteira.

☕ Spring Boot — o Java do mercado de verdade

A facul te deu a base; o mercado quer Spring. É o framework que roda em banco, fintech, seguradora e governo. Estude: criar projeto no Spring Initializr, API REST com Spring Web, Spring Data JPA + PostgreSQL, validação, tratamento de erros, autenticação com Spring Security (básico).

Spring Initializr (gerador de projeto oficial) Spring Guides oficiais (tutoriais curtos e ótimos) Michelli Brito (YouTube, Spring em PT — referência BR) Fernanda Kipper (YouTube, projetos Java/Spring práticos em PT)

Entrega: refaça em Spring Boot a API que você fez em Node na Fase 2. Mesmo domínio, outra stack. No README, compare as duas. Isso é OURO em entrevista: prova que você entende conceitos, não decoreba de framework.

🤖 IA aplicada — de usuário a construtor

Agora você constrói com IA: consumir API de LLM (é só uma API REST — você já faz isso desde o Xano), embeddings, RAG (busca em documentos + IA), function calling (IA que executa ações) e noções de agentes.

Docs da API da Anthropic (são um tutorial ótimo) Docs da OpenAI DeepLearning.AI — RAG, agentes, function calling (grátis) Hugging Face Learn (grátis)

Entrega: Projeto 4 — chatbot RAG (ex: "pergunte qualquer coisa sobre One Piece" com base de documentos, ou assistente que responde sobre PDFs enviados). Um júnior com RAG funcional no portfólio é raríssimo.

  • Simulação: peça pra IA te entrevistar (técnica e comportamental) e peça feedback brutal. Depois simule com gente de comunidade.

Por que Java vale o esforço (dados do mercado BR)

  • Bancos, fintechs, seguradoras, varejo grande e governo rodam Java. São empresas que contratam júnior em volume via programas de estágio/trainee — e a maioria dos candidatos só tem o Java raso da facul.
  • Você vai ter o que eles não têm: Spring Boot + projeto comparando stacks + SQL + Docker. Isso é perfil de júnior acima da média.
  • Java te dá tipagem e arquitetura "de gente grande" que tornam você melhor também em TypeScript e Node. As trilhas se reforçam.

Como estudar Java sem atrasar a trilha JS

  • Fases 1–2: Java fica no horário da faculdade (você já ia estudar mesmo) + 2 sessões extras de 1h por semana quando tiver prova/projeto.
  • Fase 3: inverte — Spring Boot vira o bloco principal da tarde por ~3 meses, e JS vira manutenção.
  • Sempre: conceito novo na facul → exemplo em JS no mesmo dia. Conceito novo em JS → pergunte-se "como seria em Java?".

Diferencial🤖 IA na prática — os 3 níveis

Nível 1 — Usuário (todo mundo está aqui)

Tirar dúvida e gerar código no chat. Zero diferencial. Você passa por aqui em semanas.

Nível 2 — Usuário avançado (Fases 1–2)

  • IA como professor: explicações linha a linha, exercícios sob medida, crítica de código.
  • Prompt engineering real: dar contexto, exemplos, formato de saída esperado.
  • Ferramentas de código com IA (Claude Code, Copilot, Cursor) entendendo cada linha gerada.
  • Regra de ouro: nunca commitar código que você não consegue explicar de cabeça.

Nível 3 — Construtor (Fase 3) — onde mora o tesouro

  • API de LLM: chamar Claude/GPT do seu código (Node, Java ou Python). É uma API REST como o Xano — você já sabe 70% disso.
  • Embeddings + RAG: transformar documentos em vetores e fazer a IA responder com base neles. O projeto mais pedido por empresas hoje.
  • Function calling: IA que consulta seu banco, chama suas APIs e executa ações.
  • Agentes: IA que encadeia passos pra cumprir uma tarefa.
  • Automação: n8n / scripts Python com IA — mina de freela (atendimento, triagem de e-mail, relatórios).
Anthropic — documentação da API OpenAI — documentação DeepLearning.AI — short courses grátis Hugging Face Learn n8n (automação)

Frase pra guardar: um júnior que entrega um chatbot RAG funcional sobre os documentos de uma empresa vale mais, hoje, que muito pleno que só faz CRUD. E quase nenhum júnior sabe fazer isso. Essa é a sua bandeira.

Multiplicador🌍 Inglês & Soft Skills

Inglês — o diferencial nº 1 do Brasil

Dev júnior com inglês ganha mais, acessa vaga remota internacional e aprende 2x mais rápido (o melhor conteúdo é em EN). Método: 30 min por dia, todo dia, com conteúdo técnico:

  • Meses 1–3: vídeos técnicos com legenda em inglês + app de vocabulário (Anki com termos de programação).
  • Meses 4–8: documentação só em inglês (Odin Project, React docs) + vídeos sem legenda em PT.
  • Meses 9+: vídeos sem legenda + escrever READMEs em inglês + treinar fala descrevendo seu código em voz alta.
Fireship (vídeos de 100s, perfeitos pra treino) YouGlish (pronúncia de qualquer termo em contexto real) Anki (flashcards)

Comunicação & presença

  • Escreva sobre o que aprende: 1 post/semana no LinkedIn ("aprendi X, eis o que descobri"). Em 1 ano: 50 posts. Isso é mais visibilidade do que 99% dos formandos terão na vida.
  • READMEs impecáveis: problema → solução → stack → prints → link ao vivo → o que aprendi. Recrutador lê README, não código.
  • Comunidades: Discord da Rocketseat, He4rt Developers, Frontend BR. Sua primeira vaga provavelmente vem de indicação.
  • Postura: cumprir prazo autoimposto, responder rápido, dizer "não sei, mas descubro até amanhã". Raro e valioso.
  • Documente seus erros: cada bug difícil (você já tem vários do Laurverse) vira post E resposta pra "me conte um problema difícil que você resolveu".

Fim de semana

  • Meta mínima: 1 commit + 20 min de inglês. O streak não quebra nunca.
  • Domingo, 20 min à noite: revisar a semana, marcar metas aqui no diário e planejar a próxima.

Regras anti-naufrágio

  • Celular longe nos blocos de estudo (ou app de bloqueio).
  • Dispersou? Pomodoro: 25 min foco / 5 min pausa.
  • Falhou um dia? Não compensa dobrando. Só volta no dia seguinte — culpa afunda rotina.
